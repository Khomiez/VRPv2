# วิธีการใช้ NEOS Server สำหรับตรวจสอบ VRP Solution

## ไฟล์ที่สร้างไว้

### 1. vrp_20.mod
ไฟล์แบบจำลอง AMPL (Model file) - ประกอบด้วย:
- ตัวแปรตัดสินใจ (Decision Variables)
- ฟังก์ชันเป้าหมาย (Objective Function)
- เงื่อนไขข้อจำกัด (Constraints)

### 2. vrp_20.dat
ไฟล์ข้อมูล AMPL (Data file) - ประกอบด้วย:
- ข้อมูล 20 จุด (Nodes 1-20)
- Node 20 = Depot (จุดรวมรถ)
- Node 3 = Checkpoint (จุดทิ้งขยะ)
- เมทริกซ์ระยะทาง 20x20
- ความต้องการขยะทั่วไปและรีไซเคิล
- ข้อมูลรถ 3 ประเภท (A, B, C)

## การอ่านผลลัพธ์

### ผลลัพธ์ที่สำคัญ

1. **Objective Value (Total_Cost)**: ต้นทุนรวม (บาท)
   - เปรียบเทียบกับคำตอบจาก OR-Tools

2. **ตัวแปร x[i,j,k]**: เส้นทางการเดินทาง
   - `x[i,j,k] = 1` แปลว่ารถ k เดินทางจากจุด i ไป j

3. **ตัวแปร vehicle_used[k]**: รถที่ใช้
   - `vehicle_used[A] = 1` แปลว่าใช้รถประเภท A

### ตัวอย่างการตีความ

```
Objective Total_Cost = 2400.00

x[20,1,A] = 1     # รถ A ออกจาก Depot (20) ไป จุด 1
x[1,5,A] = 1      # จาก จุด 1 ไป จุด 5
x[5,8,A] = 1      # จาก จุด 5 ไป จุด 8
x[8,3,A] = 1      # จาก จุด 8 ไป จุดทิ้ง (3)
x[3,20,A] = 1     # จาก จุดทิ้ง (3) กลับ Depot (20)

vehicle_used[A] = 1   # ใช้รถประเภท A 1 คัน
```

เส้นทาง: `20 → 1 → 5 → 8 → 3 → 20`

- **< 1%**: ดีมาก (แทบเทียบเท่ากัน)

## การแก้ไขปัญหาที่พบบ่อย

### ปัญหา: depot_node is undefined
**สาเหตุ**: ไม่ได้กำหนดค่า depot_node ในไฟล์ .dat
**วิธีแก้**: เพิ่ม `param depot_node := 20;` ในไฟล์ .dat

### ปัญหา: Infeasible
**สาเหตุ**: ไม่สามารถหาคำตอบที่ตรงตาม constraints ทั้งหมด
**วิธีแก้**:
- เพิ่ม `num_vehicles` (จำนวนรถสูงสุด)
- ตรวจสอบความจุรถว่าเพียงพอหรือไม่

### ปัญหา: เวลานานเกินไป
**สาเหตุ**: ปัญหาขนาดใหญ่
**วิธีแก้**:
- ใช้ CBC แทน CPLEX (เร็วกว่าแต่คำตอบอาจดีน้อยกว่า)
- ตั้งเวลาสูงสุดใน options

## การสร้างไฟล์สำหรับขนาดอื่น (30, 50, 80, 138 nodes)

หากต้องการสร้างไฟล์สำหรับขนาดอื่น ให้รันสคริปต์:

```bash
cd D:\projects\python\VRPv2
python neos_files\convert_to_ampl.py --size 30
```

หรือแก้ไขไฟล์ `.dat` ด้วยตนเอง:
1. เปลี่ยน `set NODES` เป็นจำนวน nodes ที่ต้องการ
2. เพิ่มเมทริกซ์ระยะทางให้ครบ
3. เพิ่มความต้องการของแต่ละจุด
